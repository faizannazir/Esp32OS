# Branding Policy

This document defines how the ESP32OS name and branding may be used.

## Purpose

ESP32OS is open source, and forks are welcome. At the same time, users should be able to distinguish official project releases from third-party derivatives.

## Allowed Without Prior Permission

- Stating factual compatibility, such as "based on ESP32OS"
- Referencing this repository in technical documentation
- Publishing forks with clear attribution and different product naming

## Not Allowed Without Prior Written Permission

- Marketing a fork or derivative as the official ESP32OS project
- Using the ESP32OS name or logo as primary product branding for a commercial offering
- Implying endorsement by ESP32OS maintainers

## Naming Guidance for Forks

Use distinct names such as:

- "<YourName> Embedded Shell (based on ESP32OS)"
- "<CompanyName> RTOS Toolkit for ESP32 (fork of ESP32OS)"

## Enforcement Notes

The source code remains licensed under MIT as defined in LICENSE. This policy applies to project identity, branding, and marketing representation.

## Contact

Open a GitHub issue in this repository to request branding permission.
